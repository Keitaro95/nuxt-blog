# Nuxt 規約準拠へのディレクトリ再構成プラン

主な乖離点：
- `components/` が atoms/molecules/organisms/templates の Atomic Design 構造
- composables が各 `features/*/composables/` に分散
- `stores/` が存在せず `useState()` で代替
- `features/` ディレクトリが独自で存在（Nuxt 規約にない）

---


## nuxt.config.ts の変更点

```typescript
// 変更前
components: [
  { path: '~/components/atoms', pathPrefix: false },
  { path: '~/components/molecules', pathPrefix: false },
  { path: '~/components/organisms', pathPrefix: false },
  { path: '~/components/templates', pathPrefix: false },
],

// 変更後 (Nuxt デフォルトの自動インポートに戻す)
// components: [...] の設定自体を削除 → components/ 配下を全て自動インポート
// ただし layout/ ui/ feature/ の prefix は自動的に付与される
// 例: components/ui/BaseButton.vue → <UiBaseButton /> or <BaseButton />
// pathPrefix: false が必要なら引き続き設定
```

Pinia 追加が必要な場合：
```typescript
modules: ['@element-plus/nuxt', '@nuxt/eslint', '@pinia/nuxt'],
```

## 推奨アプローチ（段階的移行）

1. **Phase 1**: `components/` を `layout/ / ui/ / feature/` に再編（Atomic Design を廃止）
2. **Phase 2**: 各 `features/*/composables/` を `composables/` に移動
3. **Phase 3**: `features/*/types/` を `types/` に統合
4. **Phase 4**: `@pinia/nuxt` 導入 + `useState` を Pinia store に置換
5. **Phase 5**: `features/` ディレクトリ廃止（コンポーネントは feature/ に統合済）

---

## 検証方法

- 各 Phase 完了後に `npm run dev` で動作確認
- `npm run ci`（lint + build）が通ることを確認
- 全20ページの画面表示を手動確認
- TypeScript エラーがないこと（`typescript.typeCheck: true` のため ビルド時に検出）
